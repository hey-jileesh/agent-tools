> **Purpose of this template:** Capture a business or IT objective, the problem it solves, the initiatives that realize it, current progress, and the challenges in the way. One document per objective. Reviewed at the cadence stated below.

## 1. Objective Statement

*One or two sentences. Outcome-oriented, not activity-oriented. A VP should read this and know exactly what "done" looks like.*

**Strategic alignment:** Which enterprise or divisional strategy pillar does this objective serve? Link or name it.

## 5. Contributing Initiatives

*The objective itself is not directly funded. It is realized through initiatives. Every contribution therefore falls into one of two classes, and the distinction must stay visible so leadership can act on it:*

- **Direct contribution:** The work is inside the initiative's own scope, priority, and funding. It happens by default.
- **Indirect contribution (ask):** The objective needs the initiative to do something that is *not* the initiative's priority and is *not* funded today. Each ask requires an explicit leadership decision: pursue or not — and if pursued, who pays.

### 5a. Direct Contributions — funded within the initiative

| Initiative (Epic) | How it contributes to the objective | Contribution weight* | Owner | Status | Timeline | Governance link |
|---|---|---|---|---|---|---|
| | Which success measure(s) it moves, and how | High / Med / Low | | Not started / In flight / Blocked / Done | | Link to Solution Intent / PRD / TDD |
| | | | | | | |

\* *Contribution weight = how much of the objective's success depends on this initiative. If one "High" initiative is blocked, the objective is at risk.*

### 5b. Indirect Contributions — asks requiring a funding decision

*Requests from the objective owner to initiatives (or to leadership) for work that serves the objective but sits outside the initiative's funded priorities. Nothing in this table is committed until a decision is recorded. Open decisions here also appear in Section 8.*

| # | Initiative asked | What we need it to do | Measure(s) it moves | Priority to the initiative | Funded today? | Proposed funding source | Leadership decision | Decision owner | Needed by | Status |
|---|---|---|---|---|---|---|---|---|---|---|
| 1 | | Specific, scoped ask — not "support the objective" | | Low / None | No | A: Initiative absorbs · B: Separate funding · C: Defer to next horizon | Pursue (A/B) / Defer / Decline | VP Eng / Business VP | | Open / Decided / Committed |
| 2 | | | | | | | | | | |

**Decision outcomes explained:**

| Outcome | Meaning | Consequence |
|---|---|---|
| **Pursue — initiative absorbs (A)** | Initiative takes the ask into its scope and budget | Ask moves to table 5a; initiative backlog updated |
| **Pursue — separate funding (B)** | Leadership funds the ask outside the initiative's budget | Funding source named; delivery owner assigned (may still be the initiative team) |
| **Defer** | Valid ask, wrong horizon | Carried into next horizon's objective document |
| **Decline** | Leadership accepts the objective gap | Impact recorded in coverage check below |

### Coverage Check

*An explicit, per-measure verdict on whether the **committed** contributions — 5a plus any 5b asks decided as Pursue — realize the objective. Undecided or declined asks do not count. This is an analysis, not an assertion: the verdict must state its reasoning. Every measure that is not fully Covered must have a landing place — an open decision (Section 8), an accepted risk with an owner (Section 6), or a deliberate rescoping of the measure.*
